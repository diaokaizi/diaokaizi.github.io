1 Money Robbing

A robber is planning to rob houses along a street. Each house has a certain amount of money stashed, the only constraint stopping you from robbing each of them is that adjacent houses have security system connected and it will automatically contact the police if two adjacent houses were broken into on the same night.

  1. Given a list of non-negative integers representing the amount of money of each house, determine the maximum amount of money you can rob tonight without alerting the police.
  2. What if all houses are arranged in a circle?

a)算法与伪代码

通过动态规划依次到第i家时偷与不偷所能获得的最大金额，如果偷，则最大金额为不偷上一家的最大金额加上这一家的钱，如果不偷，则最大金额为上一家偷与不偷的最大金额。

b)最优子结构与状态转移方程

假定从左边开始偷起，定义dp[i][0]表示在不触发警报的情况下偷取第i家后，最大能偷到的总金额，dp[i][1]表示不偷第i家的最大金额数。初始值：偷第一家dp[i][0] = money[i]，不偷第一家则dp[i][1] = 0

c)证明正确性

  1. 假设偷第i家，那么为了不触发警报，则前一家一定不能偷，所以dp[i][0] = dp[i - 1][1] + money[i]。

  2. 假设不偷第i家，那么dp[i][1]为前一家偷与不偷两种情况的最大值，所以

    dp[i][1] = max(dp[i - 1][0], dp[i - 1][1])。

假设一共有n家，那么最后能偷到的最大总金额为max(dp[i][0],dp[i][1])

d)时间复杂度

经过一次遍历即可得到结果，时间复杂度为O(n)

e)房屋围成一圈

如果围成一圈，那么需要额外判断第一家和最后一家，第一家和最后一家不能同时偷。分两种情况讨论：

  1. 假设不偷第一家，那么问题可以转化为在子区间[2,n]求最大金额。

  2. 假设不偷最后一家，那么问题可以转化为在子区间[1,n-1]求最大金额。

分两种情况进行计算，最后取最大的结果。

You have the following three operations permitted on a word:

Insert a character

Example:

Input: word1 = “horse”, word2 = “ros”

Output: 3 Explanation:

horse -> rorse (replace ‘h’ with ‘r’)

rorse -> rose (remove ‘r’)

rose -> ros (remove ‘e’)

定义dp[i][j]为word1的前i个字符和word2的前j个字符转化所需要的最小的操作。

使用增加字符和删除字符操作：

已知dp[i-1][j]，那么通过删除word2的第j个字符或者向word1的第i个位置添加word2的第j个字符能够得到，仅1步操作,即dp[i][j] = dp[i-1][j] + 1，同理dp[i][j] = dp[i][j-1] + 1

即dp[i][j] = min(dp[i-1][j],dp[i][j-1]) + 1

  1. 如果word1[i]!=word2[j]，那么把word1[i]修改word2[j]，或者把word2[j]修改word1[i]，需要一步操作dp[i][j] = dp[i-1][j-1] + 1

Test-and-Set Lock

最基本的锁，每个线程共用一把锁，进入临界区之前看没有有线程在临界区，如果没有，则进入，并上锁，如果有则等待。AtomicBoolean的原子特性保证一次只有一个线程可以获得到锁。

Test-Test-and-Set Lock

第一步先去检测 lock是否空闲，只有一个线程先观测到lock是空闲的，该线程才会尝试的去获取它，从而消除一部分回写操作。

Exponential Backoff Lock

给后进入自旋的线程一个延迟避让，使用避让策略为指数回退。

什么是字符串哈希

判断两个长度为的字符串是否相等，最基本的做法是逐一比较每一个字符，判断是否相等，时间复杂度为。而为了加快字符串比较的速度，我们就可以用到字符串哈希。

哈希指的是一种映射规则，一般是数字到数字的映射，而字符串哈希则表示从字符串到数字的映射。字符串哈希用来比较字符串是否相等，把比较字符串的问题转化为比较数字的问题，从而把时间复杂度降低为。

出现冲突的解决办法

冲突在字符串哈希中也会出现 ，它指的是两个不同的字符串最后计算得到的哈希值却得到了相同的结果。冲突出现时，两个字符串的哈希值相等无法真的确定这两个字符串是否相等。

如果出现了冲突，可以从下面几点来尽量的避免冲突：

  1. 修改P值，一般来说使用更大的P值在一定程度上能解决哈希冲突。
  2. 考虑使用更长的long来代替int。
  3. 两个字符串的哈希值相等后，遍历两个字符串判断是否真的是相等的，会额外花费O(n)的时间复杂度。
  4. 双哈希，字符串使用不同的字符串哈希构造方法来产生两个哈希值，只有当两个哈希值都相同的时候才表示两个字符串相等，类似还有三哈希等。
  5. 布隆过滤
