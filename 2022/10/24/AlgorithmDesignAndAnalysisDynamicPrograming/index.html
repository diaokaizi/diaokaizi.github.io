1 Money Robbing

A robber is planning to rob houses along a street. Each house has a certain amount of money stashed, the only constraint stopping you from robbing each of them is that adjacent houses have security system connected and it will automatically contact the police if two adjacent houses were broken into on the same night.

  1. Given a list of non-negative integers representing the amount of money of each house, determine the maximum amount of money you can rob tonight without alerting the police.
  2. What if all houses are arranged in a circle?

a)算法与伪代码

通过动态规划依次到第i家时偷与不偷所能获得的最大金额，如果偷，则最大金额为不偷上一家的最大金额加上这一家的钱，如果不偷，则最大金额为上一家偷与不偷的最大金额。

b)最优子结构与状态转移方程

假定从左边开始偷起，定义dp[i][0]表示在不触发警报的情况下偷取第i家后，最大能偷到的总金额，dp[i][1]表示不偷第i家的最大金额数。初始值：偷第一家dp[i][0] = money[i]，不偷第一家则dp[i][1] = 0

状态转移方程如下：

c)证明正确性

  1. 假设偷第i家，那么为了不触发警报，则前一家一定不能偷，所以dp[i][0] = dp[i - 1][1] + money[i]。

  2. 假设不偷第i家，那么dp[i][1]为前一家偷与不偷两种情况的最大值，所以

    dp[i][1] = max(dp[i - 1][0], dp[i - 1][1])。

假设一共有n家，那么最后能偷到的最大总金额为max(dp[i][0],dp[i][1])

d)时间复杂度

经过一次遍历即可得到结果，时间复杂度为O(n)

e)房屋围成一圈

如果围成一圈，那么需要额外判断第一家和最后一家，第一家和最后一家不能同时偷。分两种情况讨论：

  1. 假设不偷第一家，那么问题可以转化为在子区间[2,n]求最大金额。

  2. 假设不偷最后一家，那么问题可以转化为在子区间[1,n-1]求最大金额。

分两种情况进行计算，最后取最大的结果。

3 Largest Divisible Subset

Given a set of distinct positive integers, find the largest subset such that every pair (𝑆𝑖 , 𝑆𝑗 ) of elements in this subset satisfies: 𝑆𝑖%𝑆𝑗 = 0 or 𝑆𝑗%𝑆𝑖 = 0. Please return the largest size of the subset.

Note: 𝑆𝑖%𝑆𝑗 = 0 means that 𝑆𝑖 is divisible by 𝑆𝑗

a)算法与伪代码

由于选取的是子集，所以元素的顺序不影响结果，首先对元素从小到大进行排序，然后从左到右进行遍历。计算以当前元素作为子集中的最大元素，能够选取的最大子集大小是多少。

b)最优子结构与状态转移方程

定义dp[i]表示以s[i]为最大值的子集的大小，对于s[i]（i>0），遍历所有元素s[j],j ∈[0,i-1]，如果s[i]%s[j]=0说明s[j]能够添加到s[i]所在的子集中，计算这些可能的子集的最大值加一就是以s[j]为最大值的整除子集的大小。

初始值dp[0]=1，如果前面的所有数都不能整除s[i]，说明s[i]需要自己创建一个新的子集，dp[i]=1。

状态转移方程如下：

c)证明正确性

整除子集里面的所有数都满足s[i]%s[j]=0 或s[j]%s[i]=0那么可以推断出，集合中的每一个元素能够被集合中的比它小的元素整除。所以对于i>j如果s[i]%s[j]=0，那么s[i]一定能够被当前s[j]所在整除子集的所有数整除，于是s[i]就可以作为这个整除子集的最大元素。

由于最大整除子集不一定包含有整个列表中的最大值，所以需要对所有的整除子集进行遍历，取最大的大小。

d)时间复杂度

排序的时间复杂度是O(logn)，计算dp[i]需要遍历之前的每一个比s[i]小的数，所以求dp数组的时间复杂度为O(n2)。

Input: nums = [1,1,1,1,1], target = 3

5 Best Time to Buy and Sell Stock with Cooldown

You are given an array prices where prices[𝑖] is the price of a given stock on the 𝑖 𝑡ℎ day.

Find the maximum profit you can achieve. You may complete as many transactions as you like (i.e., buy one and sell one share of the stock multiple times) with the following restrictions:

After you sell your stock, you cannot buy stock on the next day (i.e., cooldown one day).

Note: You may not engage in multiple transactions simultaneously (i.e., you must sell the stock before you buy again).

Example:

Input: prices = [1,2,3,0,2]

Output: 3 Explanation:

transactions = [buy, sell, cooldown, buy, sell]

定义dp[i][j],j ∈[0,2]:

dp[i][0]表示到第i天时手上持有股票，能获得的最大钱数。

dp[i][1]表示到第i天时手上不持有股票，且今天没有卖掉股票，能获得的最大钱数。

dp[i][0]表示到第i天时手上不持有股票，且是今天卖掉的，能获得的最大钱数。

可推导出：

第i天时手上持有股票的可能情况是：

  1. 前一天持有股票，今天不做操作继续持有股票, 即dp[i-1][0]。
  2. 在第i天买入股票，因为只能在不处于冷冻期的状态下购买股票，即前一天不能卖掉股票，即dp[i-1][1] - prices[i]。

dp[i][0] = max(dp[i-1][0], dp[i-1][1] - prices[i])

第i天时手上手上不持有股票，且今天没有卖掉股票的可能情况是：

  1. 保持前一天的该状态, 即dp[i-1][1]。
  2. 前一天卖掉了股票，今天处于冷冻期, 即dp[i-1][2]。

6 Edit distance

Given two strings word1 and word2, return the minimum number of operations required to convert word1 to word2.

You have the following three operations permitted on a word:

Insert a character

Delete a character

Replace a character

Example:

Input: word1 = “horse”, word2 = “ros”

Output: 3 Explanation:

horse -> rorse (replace ‘h’ with ‘r’)

rorse -> rose (remove ‘r’)

rose -> ros (remove ‘e’)

定义dp[i][j]为word1的前i个字符和word2的前j个字符转化所需要的最小的操作。

使用增加字符和删除字符操作：

已知dp[i-1][j]，那么通过删除word2的第j个字符或者向word1的第i个位置添加word2的第j个字符能够得到，仅1步操作,即dp[i][j] = dp[i-1][j] + 1，同理dp[i][j] = dp[i][j-1] + 1

即dp[i][j] = min(dp[i-1][j],dp[i][j-1]) + 1

使用修改字符操作：

已知dp[i-1][j-1]，考虑两种情况：

  1. 如果word1[i]!=word2[j]，那么把word1[i]修改word2[j]，或者把word2[j]修改word1[i]，需要一步操作dp[i][j] = dp[i-1][j-1] + 1
  2. 如果word1[i]==word2[j]，那么不需要操作，即dp[i][j] = dp[i-1][j-1]

由于dp[i-1][j-1]<=dp[i][j-1]<=dp[i-1][j-1]+1，那么dp[i-1][j-1] <= min(dp[i-1][j],dp[i][j-1]) + 1，所以如果word1[i]==word2[j]，实际上不用考虑增删字符操作，直接选择dp[i][j] = dp[i-1][j-1]。
