Question1

The given string S is represented by “x” and “.”, “x” represents a pit, “.” represents a normal road. The cost of filling k consecutive pits is k+1, please obtain the maximum normal road ( It can be non-continuous. ) with budget M.

Input: S = “xxx…xxxx….xxxxx”, M = 11

Output: 16

a)算法与伪代码

  1. 统计所有的连续的坑的长度，然后按照从大到小的顺序进行排序。
  2. 依次按从大到小的顺序填补这些坑，并减去对应的预算，如果预算不足则跳过，去填补更短的坑，直到剩下的预算不能填补任何一个坑，或者坑被填完。
  3. 统计最终填补好的坑的总长度，加上原来正常的道路的长度，得到最终答案。2 5 6 9

b)证明正确性

不是最优解 比如三个坑 2 5 6 M = 9

Question2

nums is an array consisting of non-negative integers, please concatenate them in certain permutation such that they can form a smallest new integer number.

Input: nums = [1, 4, 13, 2, 25]

Output: 1132254

a)算法与伪代码

对数组进行排序。假设a和b是数组中的两个数，那么用ab表示a和b拼接后的数字。同样的，用ba表示b和a拼接后的数字。

  1. 如果ab<ba那么排序后a应该在b的前面。
  2. 如果ab>ba那么排序后b应该在a的前面。
  3. 如果ab==ba那么说明a和b可以任意排序。

将排完序的数依次组合得到的就是最小的组合数。

这里的排序使用的是选择排序，时间复杂度为$O(n^{2})$，可以使用更快的排序算法。

b)证明正确性

证明传递性：即如果有AB <= BA和BC <= CB，那么有AC <= CA。

用a表示A的位数，用b表示B的位数。所以根据AB <= BA有$A 10^{b} + B < B 10^{a} + A$，可以推导出$\frac{A}{10^{a} - 1} < \frac{B}{10^{b} - 1}$。

同理根据BC <= CB有$\frac{B}{10^{b} - 1} < \frac{C}{10^{c} - 1}$。

所以有$\frac{A}{10^{a} - 1} < \frac{C}{10^{c} - 1}$，因此可以推出AC <= CA，传递性得证。

假设按照这种比较方法对数组排序后得到的数组为[a1,a2,a3,...,an]，假设交换任意的ai和aj，0<=i<j<n，那么数组按顺序组合后的数字都会比原数组按顺序组合的数字大，因此不存在一个比[a1,a2,a3,...,an]更小的一个组合顺序，证明[a1,a2,a3,...,an]就是最小的组合顺序。

3 Largest Divisible Subset

There are n meetings, and the start time and end time of i-th meeting are si and ei. Please design a greedy algorithm to find the minimum number of meeting rooms required to arrange all meetings.

Input: [ [0, 20], [15, 20], [20, 23], [25, 30], [15, 25] ]

Output: 3

a)算法与伪代码

将数组按照会议的开始时间进行排序，并建立一个最小堆，堆中存放的是每一个房间的结束时间。具体做法是：

依次遍历数组，得到这个会议的开始时间si，从堆中取出堆顶元素priorityQueue.top()（最早结束的房间）有两种情况：

  1. si > priorityQueue.top()，说明这场会议可以安排在该房间，修改该元素为ei并重新放入堆中。

  2. si <= priorityQueue.top()，说明这场会议不能安排在所有的房间，因此需要新建一个元素ei放入堆中。

最后的房间数就是堆的元素数量。

b)证明正确性

  1. 当si > priorityQueue.top()时，假设队列中存在k个房间的结束时间都比si小，那么这个会议可以安排在这k个房间中，并且无论该会议安排哪个房间，都不影响后续的会议安排（因为后续的会议开始时间晚，当前房间能够安排这个会议，也一定能够安排后续的会议），所以这种情况不需要新建一个会议，是一个最优解。
  2. 当si <= priorityQueue.top()时，说明没有一个房间能够满足会议要求，那么必须新建一个房间。

Question4

N is a non-negative integer, remove k digits from it to obtain a new number. Find the biggest possible output number.

Input: N= 147128, k = 3

Output: 728

a)算法与伪代码

使用单调栈进行操作，具体做法是：

  1. 从左到右遍历数组的每一位：
    1. 如果k>0并且栈顶元素小于当前位数，那么栈顶元素出栈，k--，然后重复执行这一步。
    2. 当前位进栈。
  2. 如果k>0，那么栈顶元素出栈，k--，然后重复执行这一步。
  3. 最后栈中剩下的元素的组合就是最后的结果。

b)证明正确性

从数中删除k位得到最大值，那么需要尽量删掉左边的较小的数。

假设N的位数是m，的每一位为[d1,d2,d3,...,dm]，那么N删除k位后的位数为m-k。

解题的思路是删除那些比右边元素小的位，例如假设d1 < d2，如果不删除d1，那么以d2开头的m-k位数一定比d1开头的m-k位数大，因此d1必须删除，同理，如果在d2右边找到一个比d2更大的数，那么也需要把d2删除。

Question5

You are given an integer array nums. You are initially positioned at the array’s first index, and each element in the array represents your maximum jump length at that position. Return “yes“ if you can reach the last index, or “no“ otherwise

Input: nums = [2,3,1,1,4]

Output: “yes”

a)算法与伪代码

维护一个能达到的最远下标max，依次遍历，遍历过程中如果发现能够达到更远的距离，则更新max。

b)证明正确性

由于nums[i]表示的是在下标i能选择的跳跃的最大距离是nums[i]，说明如果能够到达i，那么就能够到达区间[i,i+nums[i]]的任意位置，所以维护一个最远下标m，那么m之前的任意一个位置都能够到达，因此贪心的遍历这些位置来尝试增大能够到达的最远下标。

如果可以到达的最远位置大于等于数组的最后一个下标，说明找到了答案，否则最后一个位置无法到达。

Question6

There is a number k and you can swap two digits at most once. Please design a greedy algorithm to find the maximum value you can get.

Input: k = 39748

Output: 93748

从后往前遍历每一位，维护最大位的下标（需要严格大于才更新，因为大数相同情况下，交换低位的大数比交换高位的大数要更大）。

并且遍历的过程中在这个最大位的左边找到最左边的一个比它小的下标，记录这一对下标为可能的答案。

最后得到的两个下标就是要交换的两个下标。

Question7

Given an array of integers nums, divide them to k subsets, find a division scheme to maximize the sum of the ranges (极差) (i.e., max - min) of all subsets.

Input: nums=[1, 2, 3, 4, 5], k = 3

Output: 6 (Optimal division scheme: {1, 5}, {2, 4}, {3}, sum of ranges: 4 + 2 + 0 = 6)

依次放第1大的数和第1小的数到第1组中，放第2大的数和第2小的数到第2组中……

前提是满足剩下的数>=剩下的组数，如果剩下的数<剩下的组数，说明不能放两个数。

如果每一组都放完，还有剩下的数没有放，那么这些数可以任意放，不会影响最终的结果。
